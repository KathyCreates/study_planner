# Study Planner

**Study Planner** — це веб-застосунок, який допомагає користувачам планувати й організовувати навчальні завдання, покращувати продуктивність і керувати часом. Він дозволяє створювати, редагувати й відстежувати задачі, а також надає функціональність для фільтрації та сортування завдань за різними критеріями.

## Опис

Проєкт дозволяє студентам або будь-яким користувачам планувати та стежити за їх навчальними завданнями, а також організовувати свій розклад, додаючи нові задачі з термінами виконання. Метою є спрощення процесу планування завдань і надання зручного інтерфейсу для щоденних планів.

Проєкт підтримує:
- Створення та редагування завдань
- Фільтрація завдань за датами
- Підтримка прогресу через відмітки виконаних задач
- Підтримка для роботи в оффлайн-режимі (через Service Worker)
- Підтримка темної/світлої теми

## Технології

Цей проєкт використовує наступні технології:
- **HTML** для структури веб-сторінки
- **CSS** для стилізації користувацького інтерфейсу
- **JavaScript** для функціональності, обробки подій і взаємодії з даними
- **IndexedDB** для зберігання даних локально
- **Service Worker** для підтримки оффлайн-режиму
- **PWA (Progressive Web App)** для мобільних та десктопних пристроїв
- **Web Storage API** для тимчасового зберігання даних на стороні клієнта

## Функції

### 1. **Додавання задач**
   - Користувач може додавати нові задачі, вказуючи опис і терміни виконання.
   - Доступна можливість редагувати існуючі задачі та змінювати їхній статус.

### 2. **Фільтрація та сортування**
   - Задачі можна фільтрувати за різними критеріями (наприклад, за датами, категоріями).
   - Підтримка сортування задач за термінами виконання та статусами.

### 3. **Оффлайн-режим**
   - Завдяки використанню Service Worker та IndexedDB, застосунок працює в оффлайн-режимі, зберігаючи задачі навіть при відсутності інтернету.

### 4. **Інтерфейс**
   - Інтерфейс простий і інтуїтивно зрозумілий, забезпечує швидкий доступ до всіх основних функцій.
   - Підтримка адаптивного дизайну для використання на мобільних і десктопних пристроях.

## Структура проєкту

### Директорії:

- /src
- /assets      - Статичні файли (зображення, шрифти, іконки)
- /css         - CSS файли
- /js          - JavaScript файли для логіки роботи
- /service     - Service Worker і PWA файли
- /components  - Повторно використовувані компоненти для інтерфейсу
- /utils       - Утиліти та допоміжні функції

### Файли:

- `index.html` — головний HTML файл.
- `styles.css` — основний CSS файл для стилізації інтерфейсу.
- `app.js` — JavaScript файл для управління логікою застосунку (додавання задач, обробка подій, взаємодія з локальним сховищем).
- `sw.js` — Service Worker для підтримки оффлайн-режиму.
- `manifest.json` — файл конфігурації для Progressive Web App (PWA).

## Як запустити

### Крок 1: Клонувати репозиторій

git clone [https://github.com/KathyCreates/study_planner.git](https://github.com/KathyCreates/study_planner.git)

### Крок 2: Встановити залежності
Встановіть необхідні залежності, якщо вони є (якщо використовуються пакетні менеджери, як npm або yarn):

npm install

### Крок 3: Запуск проєкту
Запустіть сервер або відкрийте файл `index.html` у браузері:

npm start

Або просто відкрийте файл `index.html` у вашому браузері.
